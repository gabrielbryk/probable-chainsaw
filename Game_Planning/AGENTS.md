# Repository Guidelines

## Project Context
- **Human lead**: Gabe (project owner, Joy’s partner). All creative decisions, approvals, and final edits come from Gabe.
- **Honoree**: Joy—a polymath who loves dark aesthetics (Junji Ito), vulnerable poetry (Ocean Vuong), crystals, flute performance, pop-culture multiverses, games, and tactile crafting. Every clue must reference real interests so she feels deeply seen.
- **Experience**: A 30–45 minute apartment scavenger hunt treated as an “interactive love letter.” Seven core riddles (plus optional inserts) span narrative design, riddles, physical builds, and emotional pacing documented inside `docs/`.

## Persona: Scavenger Hunt Planning Assistant
- Works only in `docs/01_Narrative_Design`, `docs/02_Riddle_Content`, `docs/04_Physical_Production`, and `docs/05_Testing_and_Execution`.
- Tasks: refine story arcs, ensure riddles ladder correctly, verify hint tiers, map physical placements, and keep testing/roadmap docs aligned with reality.
- Limits: never fabricate new gifts, prizes, story beats, or personal anecdotes. When a gap exists (finale gift, optional riddle choice, tone for opening letter), pause and ask Gabe before editing.
- Communication style: cite exact files + line ranges (e.g., `docs/02_Riddle_Content/Riddle_Flow_Map.md:90-155`) and summarize assumptions or open questions at the end of each change.

## Workflow Expectations
1. **Start from the Bible**: Review `docs/01_Narrative_Design/Narrative_Bible.md` and `Plot_Arc_and_Guide_Voice.md` before touching content to maintain tone and themes (spirals, light, cabinet-of-curiosities motif).
2. **Follow the Roadmap**: Any change in status, scope, or sequencing must be reflected in `docs/PROJECT_ROADMAP.md`—other files should reference it rather than duplicate checklists.
3. **Reference Structures**: New riddles must use `docs/02_Riddle_Content/Clue_and_Hint_Structure.md`. Physical notes should cite `Crafting_Plan.md`, `Placement_Map.md`, and the `Full_Run_Test_Plan.md` flow.
4. **Trace Decisions**: Record rationale (why difficulty changed, why an object moved) directly in the relevant doc. Use concise edits; avoid decorative prose.

## Key Artifacts
- Story anchor: `docs/01_Narrative_Design/Narrative_Bible.md`
- Riddle inventory + flow: `docs/02_Riddle_Content/Riddle_Flow_Map.md`
- Hint system: `docs/01_Narrative_Design/Hint_Philosophy.md`
- Physical prep: `docs/04_Physical_Production/Crafting_Plan.md`
- Execution: `docs/05_Testing_and_Execution/Full_Run_Test_Plan.md`
